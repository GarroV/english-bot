## Средний приоритет

### Статистика для admin
Команда `/stats` — количество зарегистрированных пользователей, заданий в кэше, заданий за последние 7/30 дней.

### Отзыв инвайт-кода
Команда для admin — пометить **неиспользованный инвайт** как недействительный (добавить поле `revoked_at` в `eb_invitations`). Отдельно от отзыва доступа у уже зарегистрированного пользователя (тот реализован: `/revoke <telegram_id>` / `/restore`, 2026-07-03). Занять другое имя команды, чтобы не конфликтовать с `/revoke <telegram_id>`.

### Отзыв доступа — хвосты (feature от 2026-07-03)
Мягкий отзыв доступа (`/revoke`/`/restore`, флаг `disabled_at` в `eb_users` + `folio_users`, блок Folio через RLS-null) реализован. Открытые продолжения:
- **Полная инвалидация Supabase refresh-токенов при отзыве.** Сейчас блок идёт через RLS-null (`folio_current_workspace_id()` → null → все воркспейс-запросы пусты), что покрывает доступ к данным немедленно. Но выданный access-token живёт до истечения — правда, RLS-null уже делает его бесполезным для данных воркспейса. Для строгого мгновенного логаута нужно принудительно отозвать сессии Supabase Auth (admin API `signOut(scope: 'global')` / revoke refresh-токенов) при `/revoke`.
- **Судьба данных отключённого воркспейса** — архив / экспорт / передача другому репетитору. Сейчас данные просто сохраняются нетронутыми и невидимы владельцу; политика долгосрочного хранения не определена.
- **UI отзыва в Folio-админке** — сейчас отзыв только командой бота (`/revoke <telegram_id>`). Дать super-admin кнопку «отозвать/восстановить доступ» в веб-админке Folio.
- **Кабинеты учеников отозванного репетитора не блокируются** (security-review 2026-07-03, LOW). Кабинет ученика (`/s/<token>`) — service-role по `cabinet_token`, не зависит от `folio_current_workspace_id()` / `folio_users.disabled_at`, поэтому при отзыве репетитора его ученики продолжают видеть/сдавать ДЗ и писать в чат. По дизайну токен-модели, не регрессия. Если отзыв должен отрезать и учеников — нужен явный каскад (проверять `disabled_at` владельца воркспейса в cabinet-экшенах). Продуктовое решение.

### Rate limiting
Ограничение: не более N генераций в час на пользователя. Считать по `eb_assignments.created_at` или отдельной таблицей. Защита от случайных спамов.

### Webhook self-registration при деплое
Скрипт или Makefile-таргет, который после `supabase functions deploy` автоматически вызывает `setWebhook`. Сейчас надо делать вручную curl-ом.

## Низкий приоритет

### Поиск по своим заданиям
`/search <запрос>` — семантический поиск по `eb_assignments` пользователя, возвращает топ-3 похожих задания.

### Повторная генерация из POST_GENERATION
Кнопка "🔄 Сгенерировать другой вариант" в `POST_GENERATION` — регенерация с теми же параметрами, минуя кэш. Сейчас нужно начинать заново через "🆕 Новое задание".

### Поддержка других языков обучения
Параметр "язык" (немецкий, французский, испанский) — добавить в `ClarifyingParams` и визард (`buildWizardMessage`/`handleWizardStep`). Промпты адаптировать или сделать отдельные.

### Экспорт в DOCX
Альтернатива PDF для пользователей, которым нужно редактировать задание. Библиотека `docx` для Deno/Node.

### Папки / теги для заданий
Пользователь может отметить задание тегом или добавить в папку для организации архива.

# Folio — Бенто-дашборд репетитора (design spec)

> Дата: 2026-06-30 · Статус: дизайн утверждён владельцем (макет-прототип), готов к плану реализации.
> Прототип: Artifact `bento-v3` (тёмная/тёплая тема, раскрытые правые блоки).

## 1. Обзор и цель

Сейчас `/[locale]/dashboard` — заглушка: показывает только роль. Превращаем её в **командный центр репетитора** в формате бенто — обзор дня + быстрая генерация задания, всё на одном экране. Single-tutor (не команда). Структура вдохновлена Swarm Brain, адаптирована под домен Folio.

**Не цель:** новые сущности БД (кроме опционального лога генераций), монетизация, кабинет ученика.

## 2. Поверхность

Заменяет `src/app/[locale]/(app)/dashboard/page.tsx`. Остальные роуты (`/schedule`, `/billing`, `/homework`, `/admin`) — без изменений; дашборд на них **ссылается**.

## 3. Раскладка (утверждена)

**Шапка** (через весь экран): 🦉 Folio + «Кабинет репетитора» слева; справа — `📋 История` (архив сгенерированных заданий), быстрые действия `⊞ Ученики` (админка ростера), `+ Занятие`, `+ Оплата`, и **переключатель темы** (светлая/тёмная).

**Бенто, 3 колонки** (≤1080px — стопкой):
- **Слева — «Занятия сегодня».** Список занятий на сегодня (время, ученик/группа, уровень, «сейчас»-метка). **Клик по шапке карточки → `/schedule`** (неделя/месяц).
- **Центр (большой) — «Сгенерировать задание» + вычитка.** AI-инпут (тема/уровень/тип) → черновик → правка («что поправить») → действия: Сохранить шаблон / PDF / Заново / Назначить ученику. Зеркалит флоу бота (генерация → POST_GENERATION → правка).
- **Справа — два раскрытых мини-блока:** «Домашки» (на проверку / просрочено + список) и «Долги» (сумма + должники). Раскрыты по умолчанию; клик по шапке — свернуть.

## 4. Данные — переиспользуем существующее (без новых таблиц)

| Блок | Источник (существует) |
|---|---|
| Занятия сегодня | `lib/lessons/queries.ts` `listLessonsInRange` (фильтр на сегодня по TZ) |
| Долги | `lib/billing/queries.ts` `listBalances` (только `balance > 0`) |
| Домашки на проверку / просрочено | `lib/homework/queries.ts` `listAssignments` (`status='assigned'` + `due_date < today` = просрочено; `submitted` = на проверку) |
| Ученики (быстрое действие) | `lib/students/queries.ts` `listStudents` |

Дашборд — серверный компонент: грузит данные параллельно, отдаёт тонким клиентским плиткам. Бизнес-логика в `lib/`, не в компонентах (правило проекта).

## 5. Центр: генерация + вычитка

- Генерация и сохранение — **переиспуют существующее** `lib/homework/` (`callGenerate` → `folio-generate`, `saveTemplate`, `assignTemplate`).
- **Вычитка (новое):** движок `_shared/generate.ts` уже умеет `applyEdit`, но Edge Function `folio-generate` сейчас выставляет только `generateModuleContent`. **Расширить `folio-generate`** действием `edit` (вход: текущий контент + правка → `applyEdit`), под тем же секретом `FOLIO_GENERATE_SECRET`. Веб-обёртка — `lib/homework/generate.ts` `callEdit`.
- **Единая точка генерации (хук на будущее):** все вызовы генерации/правки идут через один модуль `lib/homework/generate.ts`. Сюда позже встанет проверка лимита/кредитов (issue #23) — без переделки дашборда.

## 6. Шапка: быстрые действия (переиспуют существующие диалоги)

- `+ Занятие` → `LessonDialog` (create).
- `+ Оплата` → диалог `recordPayment` (`lib/billing/actions.ts`).
- `⊞ Ученики` → управление ростером (существующие `StudentsPanel` / `StudentForm`) — открыть как диалог/панель с дашборда.
- `📋 История` → архив сгенерированных заданий (библиотека шаблонов) — переиспует `lib/homework/queries.ts` `listTemplates`; открыть как панель/диалог либо ссылка на `/homework`.
- **Переключатель темы** — см. §8.

## 7. Компоненты (новые)

- `(app)/dashboard/page.tsx` — серверный загрузчик данных + раскладка.
- `DashboardBento.tsx` — сетка-обёртка (responsive).
- `TodayLessons.tsx`, `GeneratePanel.tsx` (с черновиком/вычиткой), `MiniBlock.tsx` (раскрываемый, ×2), `HeaderActions.tsx`.
- Файлы ≤ ~250 строк; логика данных — в server-компоненте/`lib`, не в клиентских плитках.

## 8. Тема — светлая + тёмная (обе в MVP, продумать тщательно)

- **Палитры уже есть** в `globals.css`: `:root` (тёплая светлая) + `.dark` (тёмная). Дашборд использует **только токены** (`--primary`, `--coral`, `--card`, `--sidebar*`, …) → обе темы получаются без хардкода цветов.
- **Механизм:** `next-themes` (уже в зависимостях). `ThemeProvider` (`attribute="class"`, `defaultTheme="system"`, `enableSystem`) — в корневом `[locale]/layout.tsx`; `suppressHydrationWarning` на `<html>` → без вспышки темы при загрузке (критично на OpenNext/CF: провайдер клиентский, класс применяется до paint встроенным скриптом next-themes).
- **Переключатель** в шапке: система / светлая / тёмная; выбор персистится (next-themes → localStorage). Доступность: кнопка с `aria-label`, видимый focus.
- **Проверить тёмную сквозь весь UI**, не только дашборд: сайдбар, диалоги (`LessonDialog`/`StudentForm`/оплата), таблицы — токены для них в `globals.css` есть, но визуально пройтись и поправить контраст где нужно.
- Семантика цвета (обе темы): coral = долги/просрочка, amber = на проверку, green = получено — **отдельно** от teal-акцента.

## 9. Идентичность

- Сова-знак (плоская, teal) в лого шапки — заменяет дефолтный фавикон create-next-app (это закрывает запрос на тематическую иконку). Источник стиля — аватар бота, упрощённый. Реализация — `app/icon.svg` (Next.js App Router).

## 10. Вне MVP (полировка/будущее)

- Визуальная полировка (атмосфера, анимации, финальная сова).
- Дайджест за неделю — **не делаем** (решено: смысла нет; вместо него — кнопка «История», §6).
- **Контроль генераций / кредиты — issue #23** (учёт → лимиты/планы); сейчас только заложен хук (§5).

## 11. Тестирование

- Unit: чистая логика «просрочено» (`due_date < today`), фильтр «сегодня» по TZ, агрегаты — `vitest`.
- Smoke: `scripts/smoke-render.mjs /ru/dashboard` рендерится без ошибок.
- Type-check/build: `npx tsc --noEmit`; полноценная Next-сборка через `npm run build` перед PR.

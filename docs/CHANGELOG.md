### feat: квота генераций на воркспейс + учёт веб-генераций (#75, хвост #23)

Супер-админ теперь управляет лимитом генераций каждого репетитора. Колонка `folio_workspaces.generation_quota` (NULL = безлимит, миграция `20260708220000`), остаток считается на чтении: quota − количество module-вызовов в `eb_llm_usage` (бот по telegram_id владельца + folio по workspace_id) — канон расчёта в новом `_shared/quota.ts`, общий для бота и `folio-generate`. Enforcement до платного вызова: бот — в `generateAndSend` (сообщение «Лимит генераций исчерпан…»), веб — `folio-generate` возвращает 402 `quota_exceeded` (Folio показывает понятную ошибку). Проверка fail-open: сбой чтения квоты не ломает генерацию (мягкий биллинг-контроль). Попутно закрыт хвост #23: `generateHomework` передаёт `workspace_id`, и `folio-generate` теперь пишет учёт расхода веб-генераций в `eb_llm_usage` (source='folio') — они видны в статистике админки и считаются квотой. Выдача: в админке в статистике воркспейса блок «Квота генераций» — остаток, «+ генерации» (первый грант поверх безлимита даёт лимит «израсходовано + N»), «Безлимит». Правки/teacher-guide/itemize квотой не считаются — только генерация модуля.

### feat: качество генерации — уровневый ориентир CEFR, грамматический фокус, опора для говорения

Аудит генерации (сверка с эталонным паком «Speaking Club», issue #58/#59) выявил три системные проблемы: модель дрейфовала по уровню (в C1-задании встречались ошибки уровня A2 в Error correction), грамматическая тема не выдерживалась (запрос «Past Continuous» давал модуль, где целевую конструкцию отрабатывала лишь часть заданий), а говорение шло без опоры (голые Discussion-вопросы). Правки в общем движке `_shared/generate.ts` (бот + Folio):

- **Уровневый ориентир** — словарь `LEVEL_GUIDE` (A2–C2: лексика/грамматика/длина предложений) подставляется в каждый промпт как `{LEVEL_DESC}`; промпт обязывает держать уровень и в тексте, и в каждом задании. Сложность ошибок Error correction привязана к уровню (для B2+ — времена/предлоги/коллокации, не элементарное согласование).
- **Грамматический фокус** — если запрос это грамматическая тема: в Reading текст насыщается целевой конструкцией + добавляется задание Grammar practice (выбор формы), Gap fill целится в конструкцию; в Translation (предложения) КАЖДОЕ предложение требует целевой конструкции, блоки — её разные употребления.
- **Опора для говорения** (из эталона) — в Reading добавлен блок Agree or disagree (3–4 мнения персонажей с именем/возрастом), yes/no-вопросы Discussion получают добивку «Why?/Give an example», в вопросы вшивается лексика словарного блока, 1–2 вопроса локализованы под жизнь ученика в России.
- **Честные ключи** — Teacher's Guide больше не выдумывает ответ на вопрос-мнение/открытый вопрос без ответа в тексте: ставит прочерк с пометкой.
- **Сквозная нумерация Task** без пропусков + строка-инструкция ученику под каждым заданием; в Vocabulary исправлено неоднозначное Collocations (теперь предложение с пропуском + два варианта в скобках).
- **Детекция модуля** (`module_detect.ts`) — названные грамматические темы (времена, conditionals, passive, reported speech, рус. «паст континиус» и т.п.) теперь роутятся в TRANSLATION_SENTENCES; голые `past`/`future` не триггерят (частые слова тем). Добавлены тесты.

Изменение затрагивает обе поверхности (бот и Folio через `folio-generate`). Миграций БД нет.

### feat: отзыв доступа у действующего пользователя (мягкий, обратимый) — бот + Folio

Раньше отозвать можно было только неиспользованный инвайт; после регистрации способа отключить пользователя не было. Добавлен админ-отзыв, который мягко (обратимо, без удаления данных) блокирует и бота, и Folio, включая активную веб-сессию.

**Модель** — двусторонний флаг `disabled_at` (null = активен): ставится одновременно в `eb_users` (гейт бота) и, если Telegram связан с Folio, в `folio_users` (гейт Folio). Две additive-миграции: `20260703140000_eb_users_disabled.sql` (колонка `eb_users.disabled_at`), `20260703140001_folio_users_disabled.sql` (колонка `folio_users.disabled_at` + `folio_current_workspace_id()` дополнена `and disabled_at is null` — сигнатура/атрибуты не менялись, только WHERE).

**Каскад блокировки:** (1) бот — `isAllowed` теперь требует `disabled_at is null`; гейт различает «отозван» (новая `isDisabled` → сообщение «доступ отозван») и «не зарегистрирован»; (2) новый логин Folio — `confirmFolioLogin` в начале проверяет оба флага и возвращает новый исход `disabled`, не подтверждая токен; (3) активная Folio-сессия — блокируется немедленно через RLS: отключённый репетитор не резолвит воркспейс (`folio_current_workspace_id()` → null) → все RLS-запросы пусты, Folio деградирует чисто (пустые состояния / редирект по `!user`), JWT принудительно не инвалидируется. Defense-in-depth: `resolveFolioWorkspace` (мост бот→веб) тоже возвращает null для отключённых.

**Админ-команды** (только `ADMIN_USER_ID`): `/revoke <telegram_id>` → `revokeAccess` (отвечает, что реально отключено: бот / Folio / оба / не найдено), `/restore <telegram_id>` → `restoreAccess` (зеркально). Аргумент разбирается чистой `parseTargetTelegramId` (положительное целое). `/users` помечает отключённых « — 🚫 отключён». Перерегистрация по новому инвайту (`registerUser` с `disabled_at: null`) реактивирует бот-доступ. Все update — только по `WHERE` (telegram_id / id), без DELETE.

Folio-код не тронут: приложение уже деградирует чисто при null-профиле (страницы `(app)` проверяют `!user` и редиректят; layout `getSuperAdmin()` → null; воркспейс-скоуп server-actions возвращают `{ok:false}` по `!workspaceId`) — падений нет. deno check (index.ts + весь граф) 0; deno test `lib/` — 46 passed (9 новых на `parseTargetTelegramId`). Миграции не применялись, деплоя нет — по заданию.

## Диагностика и инциденты

### 2026-05-25 — бот не отвечал после пополнения баланса Anthropic

**Причина:** секрет `ANTHROPIC_KEY` не был задан в Supabase (в продакшене был только `OPENAI_API_KEY` от старой версии). `lib/claude.ts:4` читает `Deno.env.get("ANTHROPIC_KEY")` — получал `undefined`, SDK падал при инициализации.

**Решение:**
```bash
supabase secrets set ANTHROPIC_KEY=<ключ>
supabase functions deploy english-bot --no-verify-jwt
```

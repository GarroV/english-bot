# Folio M8 — Кабинет ученика (design)

> Статус: спека (2026-07-01). Источник — брейншторм 2026-07-01.
> Веха M8. Хэнд-мейд документ; при реализации держать в соответствии с кодом.

## 1. Обзор

Персональный веб-кабинет ученика, где он видит **свои домашние задания** (актуальные и пройденные, с содержимым, PDF и комментарием учителя) и **ближайшие занятия**. Доступ — по персональной ссылке-токену, **без логина и без Telegram** (в РФ Telegram нестабилен; веб-ссылка надёжнее и заодно служит каналом доставки — ученик держит её в закладке). Read-only, с единственным действием ученика — отметить задание «Я сделал».

**Кто пользуется:** ученик репетитора (роль в БД пока не заводится — ученик остаётся строкой `folio_students`; токен и есть его «вход»). Репетитор выдаёт ссылку и ведёт комментарии/проверку в существующих экранах.

**Проблема, которую закрывает:** петля «репетитор назначил → ученик получил → сделал → репетитор проверил и прокомментировал» сейчас не замкнута — ученик вообще ничего не видит. Кабинет замыкает её без зависимости от Telegram/email.

## 2. Решения (из брейншторма)

| Ось | Решение | Почему |
|---|---|---|
| Доступ | Веб-ссылка с токеном, без логина, ротируемая | Не зависит от Telegram (блокировки в РФ); нет auth/email-инфры; ссылка = канал доставки |
| Интерактив | Просмотр + «Я сделал» (`assigned→submitted`) | Замыкает петлю дёшево; статусы уже есть в БД |
| Содержимое ДЗ | Текст на экране **+ скачивание PDF** | Читать онлайн + печатать бланки; PDF переиспользует готовый `pdf.ts` бота |
| Расписание | Read-only «ближайшие занятия» | Данные и доступ уже есть; кабинет только с ДЗ тонковат |
| Деньги/баланс | **Вне MVP** | Деньги чувствительнее; показ долга ученику — отдельный UX-вопрос |

## 3. Модель доступа

- Публичный роут **`/[locale]/s/[token]`** — вне tutor-оболочки `(app)` (нет TopNav/гейта). В `middleware.ts` добавить `s` в `PUBLIC_FIRST_SEGMENTS` (рядом с `login`/`invite`).
- `token → student_id` резолвится **service-role** admin-клиентом (как `folio_login_tokens`: RLS не для токена, роут ходит `createAdminClient()`); сам токен и есть аутентификация-capability.
- Токен: `randomBytes(24).toString("base64url")`, хранится в `folio_students.cabinet_token` (unique). **Ротируемый**: репетитор перегенерирует → старая ссылка мгновенно мертва. Без срока (закладка).
- Все запросы кабинета/действий **строго скоупятся по `token→student_id`**; `student_id` из ввода не принимается. RLS рабочих таблиц не ослабляется.

## 4. Данные (миграция, всё additive)

Файл `supabase/migrations/<ts>_folio_student_cabinet.sql`:

```sql
alter table folio_students add column cabinet_token text unique;              -- персональный токен кабинета
alter table folio_homework_assignments add column tutor_comment text;         -- комментарий учителя (один на задание)
alter table folio_homework_assignments add column submitted_at timestamptz;   -- когда ученик нажал «Я сделал»
```

Новых таблиц нет. Статусы `folio_homework_assignments.status` уже: `assigned → submitted → reviewed`. Занятия ученика — из `folio_lesson_students → folio_lessons` (существуют). Обновить `Folio/docs/DATA_MODEL.md`.

## 5. Поверхности

| Поверхность | Технология | Назначение |
|---|---|---|
| `/[locale]/s/[token]` | Next route (публичный) | Кабинет ученика: ДЗ + расписание |
| `folio-homework-pdf` | Supabase Edge Function (Deno) | PDF задания по токену (переиспользует `pdf.ts`) |
| Карточка ученика (раздел «Ученики») | существующий экран | Кнопка «Кабинет ученика» — сгенерировать/ротировать + копировать ссылку |
| Домашки → `AssignmentsList` | существующий экран | Комментарий + «Проверено» (`reviewed` + `tutor_comment`) |

## 6. Компоненты и файлы

**Новый модуль `Folio/src/lib/cabinet/`:**
- `queries.ts` — `getCabinet(token)`: резолв токена → `{ student, assignments, lessons }`, всё скоуплено; группировка ДЗ (актуальные=`assigned|submitted`, пройденные=`reviewed`); занятия — ближайшие (+ пара прошедших), **без имён со-учеников** (только тип solo/group).
- `actions.ts` — `markSubmitted(token, assignmentId)`: `"use server"`, скоуп по `token→student`, проверка что задание принадлежит ученику → `status='submitted', submitted_at=now()` (только из `assigned`).
- `token.ts` — генерация токена (общая с tutor-экшеном ротации).

**Роут/UI:**
- `Folio/src/app/[locale]/s/[token]/page.tsx` — server component: `getCabinet`, рендер (или «ссылка недействительна» при неизвестном токене).
- `Folio/src/app/[locale]/s/[token]/StudentCabinet.tsx` (`"use client"` для «Я сделал») + презентационные карточки (ДЗ, занятие).
- `Folio/src/middleware.ts` — `PUBLIC_FIRST_SEGMENTS += "s"`.

**Tutor-сторона (существующие модули):**
- `Folio/src/lib/students/actions.ts` — `ensureCabinetToken(studentId)` / `rotateCabinetToken(studentId)` (гейт: своя рабочая область).
- Карточка ученика — кнопка + показ/копирование ссылки (как инвайты в админке).
- `Folio/src/lib/homework/assignments.ts` — `reviewAssignment(id, comment)`: `reviewed` + `tutor_comment` (расширяет существующую логику статусов).
- `AssignmentsList` — поле комментария + «Проверено».

**PDF-функция `supabase/functions/folio-homework-pdf/index.ts`:**
- Публичная (`--no-verify-jwt`), `GET ?token=<t>&a=<assignmentId>`.
- Скоуп: `token→student → assignment(принадлежит ученику) → template.content`. Генерит PDF через `../english-bot/lib/pdf.ts` (`generatePdf`), отдаёт `Content-Disposition: attachment`.
- Не доверять `a` вслепую — проверять принадлежность ученику из токена.

## 7. Поток данных

```
Репетитор: назначил ДЗ (assigned) / выдал ссылку кабинета ученику
   → Ученик открывает /s/<token>: видит «Актуальные» (assigned|submitted) + «Пройденные» (reviewed) + ближайшие занятия
   → «Скачать PDF» → folio-homework-pdf (по токену) → PDF
   → «Я сделал» (assigned→submitted) → у репетитора всплывает в «на проверку» (дашборд)
   → Репетитор в Домашках: комментарий + «Проверено» (reviewed + tutor_comment)
   → Ученик видит комментарий в «Пройденных»
```

## 8. Безопасность

- **Ссылка = доступ** (длинный случайный токен). Данные некритичны (свои задания, имя, времена занятий). Митигация утечки — ротация токена репетитором.
- **Приватность групп:** в расписании — только тип «индивидуальное/групповое», **никаких имён со-учеников** (принцип M8/Groups).
- Все чтения/действия скоупятся по `token→student_id`; `student_id`/`assignment_id` из ввода перепроверяются на принадлежность. Service-role только в скоупнутых точках (роут/функция), RLS-таблицы не ослабляем.
- CSP/security-заголовки (#11, middleware) применяются к публичному роуту автоматически.
- «Я сделал» и PDF — capability по токену; guessing защищён энтропией токена (192 бита).

## 9. Вне MVP (заложить потом)

- Баланс/долг ученику (деньги — отдельное UX-решение).
- Полный интерактив: сдача ответов/файлов в кабинете, авто-проверка.
- Per-user таймзона (расписание пока МСК).
- Логин ученика (email magic-link / Telegram) вместо ссылки-токена — апгрейд, если понадобится.
- Пуш-доставка (Telegram/email) — кабинет-ссылка её замещает в MVP.

## 10. Тестирование

- Unit (`lib/cabinet`): группировка ДЗ (актуальные/пройденные), фильтр занятий (ближайшие, без со-учеников), резолв токена (валидный/неизвестный/ротированный).
- `markSubmitted`: только `assigned→submitted`; чужой assignment по токену — отказ.
- PDF-функция: token→assignment скоуп (чужой `a` → 404/403).
- Смоук на проде: открыть `/s/<token>` реального ученика, «Я сделал», «Скачать PDF».

## 11. Порядок сборки

1. Миграция (колонки) + `DATA_MODEL.md`.
2. `lib/cabinet` (queries/actions/token) + unit-тесты.
3. Публичный роут `/s/[token]` + UI (ДЗ + расписание) + middleware.
4. `folio-homework-pdf` Edge Function (переиспользует `pdf.ts`).
5. Tutor-сторона: кнопка ссылки в карточке ученика; комментарий+«Проверено» в Домашках.
6. i18n-ключи (ru/en) для всех новых текстов.
7. Деплой (миграция → Folio → функция) + смоук.

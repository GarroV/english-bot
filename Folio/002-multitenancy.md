# ADR-002: Мультитенантность через workspace_id + RLS

Date: 2026-06-04
Status: Accepted

## Контекст

Folio должен поддерживать нескольких репетиторов в будущем. На старте — один.

## Решение

Shared database, isolation через `workspace_id` на каждой таблице + RLS политики Supabase. Воркспейсы создаёт Super Admin вручную.

Не выбрали schema-per-tenant: избыточно для текущего масштаба, сложнее в обслуживании.
Не выбрали database-per-tenant: слишком дорого и сложно на старте.

## Последствия

- RLS обязателен на каждой таблице — это правило в CLAUDE.md
- Миграции добавляют `workspace_id` с первого дня, даже если репетитор один
- Онбординг второго репетитора = создание записи в workspaces + инвайт, без изменений схемы

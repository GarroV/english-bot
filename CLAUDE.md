# English Bot — Инструкции для Claude

## Стек

- **Runtime:** Deno / Supabase Edge Function
- **Язык:** TypeScript
- **LLM:** Anthropic claude-sonnet-4-20250514 (`lib/claude.ts`)
- **БД:** Supabase Postgres + pgvector (`lib/db.ts`)
- **Тесты:** `deno test supabase/functions/english-bot/lib/ --allow-env`
- **Деплой:** `supabase functions deploy english-bot --no-verify-jwt`

## Стиль кода

- Короткий комментарий над каждой функцией (одна строка — что делает)
- Без лишних абстракций, без feature flags
- Ошибки валидируются только на границах (Telegram update, Supabase response)

## Правила обновления документации

**При каждом изменении кода обязательно обновлять документацию:**

### CHANGELOG.md (`docs/CHANGELOG.md`)

Добавлять запись при **каждом коммите** с изменением кода:

```markdown
## YYYY-MM-DD

### тип: краткое описание

Один-два абзаца: что изменилось и почему. Для fix — симптом и причина.
```

Типы: `feat` (новая функциональность), `fix` (исправление бага), `refactor` (рефакторинг без изменения поведения), `chore` (инфраструктура, зависимости).

Новые записи добавлять **в начало файла**, под заголовком `# Changelog`.

### BOT.md (`docs/BOT.md`)

Обновлять при изменении **архитектуры**:

- новый handler или lib-модуль → добавить в раздел "Файловая структура"
- новое состояние сессии → обновить раздел "Машина состояний"
- новый тип модуля → обновить раздел "Типы модулей"
- новая таблица в БД → обновить раздел "База данных"
- новая команда или env-переменная → обновить соответствующие таблицы

### BACKLOG.md (`docs/BACKLOG.md`)

- При реализации фичи из беклога — **удалить** её из `BACKLOG.md`
- При обнаружении нового технического долга — добавить в раздел "Технический долг"
- Новые идеи добавлять в подходящий раздел по приоритету

### docs/README.md

Обновлять при изменении команд бота, env-переменных, шагов деплоя.

---

## После каждого изменения кода

1. Закоммитить все изменения
2. Задеплоить: `supabase functions deploy english-bot --no-verify-jwt`

Не считать задачу выполненной пока не задеплоено.

## Миграции БД

При изменении схемы создавать новый файл в `supabase/migrations/` с именем `YYYYMMDDHHMMSS_description.sql`. Применять через `supabase db push` или `supabase migration up`.

## Инвайт-система

Новых пользователей добавлять через `/invite` в боте (только admin). Прямая вставка в БД не рекомендуется — обходит логику `useInvite` и `registerUser`.
